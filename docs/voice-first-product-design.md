# 纯语音绘图产品设计与实现记录

## 1. 产品目标

用户在创作过程中不依赖鼠标或键盘，仅通过自然语言语音完成绘制、修改、撤销、重做、清空和连续补充内容。

运行前可由部署者通过 `LLM_PROVIDER` 选择 `openai` 或 `bailian`，并通过 `OPENAI_API_KEY` 或 `DASHSCOPE_API_KEY` 配置模型服务；未显式指定 Provider 时，服务端会根据已配置的百炼 Key 自动选择百炼。创作者也可以通过右上角设置面板临时覆盖 Provider、模型和 API Key，其中 API Key 仅保存在当前浏览器会话中，留空时继续使用服务端配置，服务端 trace 会递归脱敏敏感字段。主界面不提供文本输入或素材操作按钮，素材候选仅作为语音选择过程的只读反馈。首次麦克风授权是浏览器强制的安全边界，不属于创作流程。

设计重点：

- 指令理解准确且能容忍口语停顿、填充词和少量识别噪声。
- 高频控制指令尽量在本地立即执行，降低语音到画布变化的延迟。
- 模型处理期间继续接收语音，按顺序执行，避免指令丢失。
- 复杂请求能够拆解为多次绘图操作，并在内容不足时继续执行。
- 每次操作通过语音反馈确认结果，错误时给出可恢复提示。

## 2. 架构

```text
麦克风
  -> Web Speech API 持续识别
  -> 文本标准化与噪声过滤
  -> 本地控制命令 / 远程绘图命令队列
     -> 本地命令直接操作 Fabric.js
     -> 远程命令携带画布状态与最近对话历史
        -> DrawingModeRouter
        -> AgentRunner 拆解并调用绘图工具
        -> Fabric.js 代码返回前端执行
  -> SpeechSynthesis 语音反馈
```

### 2.1 指令准确性与容错

- 对全角字符、标点、空白和常见开头填充词进行标准化。
- 丢弃“嗯、你、然后”等低信息识别结果，短绘图名词如“圆”仍正常提交。
- 将短暂停顿内被 Web Speech API 拆开的 final 片段合并为一条指令，并修复“带你来画”“话术”“一棵房子”等常见绘图识别错误。
- 本地命令采用整句匹配，避免把“清空画布然后画一个圆”错误识别为立即清空。
- 清空画布采用二次语音确认，用户需继续说“确认清空”或“取消”。
- 最近 8 条 user/assistant 对话随绘图请求发送，支持“在它旁边再画一个”等连续指令。
- 服务端再次过滤历史角色、长度与数量，避免客户端历史污染提示词。

### 2.2 响应延迟

- “撤销、重做、清空确认”不调用 LLM，直接操作本地画布。
- “停止当前绘图”使用 `AbortController` 立即停止前端等待、阻止结果应用到画布，并清空排队指令。
- 识别保持持续运行；模型执行期间的新指令进入 FIFO 队列，不被丢弃。
- 复杂绘图提示模型批量调用互不依赖的工具，减少模型往返次数。
- API 返回端到端 `latency_ms`，前端在语音反馈中播报耗时，便于持续测量。
- TTS 播报期间暂时挂起识别，避免系统把自己的播报识别成新指令。

### 2.3 复杂指令执行

- 路由器识别“场景、插画、细节丰富、背景/前景”等复杂请求。
- 多主体且包含多个“旁边/附近/前后”空间关系的口述会直接进入复杂场景模式。
- Agent 按背景、中景、前景、细节分层构图，并为元素分配不同坐标，避免全部叠在中心。
- 复杂模式至少获得 12 轮执行预算。
- 少于 6 次成功绘图操作时，Agent 不允许提前宣布完成。
- 当用户要求“3D、立体、透视、正常结构、复杂结构”时，单体对象也进入复杂矢量模式，不再退化为普通图标搜索。
- `draw_vector_composition` 接收模型生成的分层 SVG，在安全清洗后导入 Fabric.js，支持曲线路径、渐变、遮罩、阴影层和高光层。
- `draw_perspective_vehicle` 为透视汽车提供稳定的语义级绘图能力，生成可编辑的三分之四视角车身、前脸、玻璃、透视轮组、灯光、接缝、阴影和高光，避免模型退化成矩形与圆形拼装。
- `draw_concentric_circles` 用于同心圆、同星圆、靶心、套圆等明确“共用圆心”的简单几何，避免把内外层误画成互不相干的两个圆。
- 对完整的复杂单体构图要求至少 10 个可见矢量元素；透视汽车应包含车身轮廓、车舱、玻璃、前后空间面、不同尺度车轮、灯光、接缝、地面阴影和高光。
- 多段 Fabric.js 工具代码使用独立作用域，避免批量操作时变量重复声明。

### 2.4 常见绘画软件操作映射

调研主流绘画/设计工具后，指令词汇按操作族进入系统提示和设计清单：

- 基础绘制：画笔、铅笔、橡皮、涂抹、克隆、模糊、锐化、加深/减淡、油漆桶、填充、渐变、颜色替换。当前以矢量路径、颜色修改和分层 SVG 近似承载，真实光栅笔刷类工具尚未实现。
- 矢量绘制：形状、直线、椭圆、多边形、折线、钢笔、自由钢笔、曲率钢笔、贝塞尔路径、锚点、描边、填充、文字。当前由 `draw_circle`、`draw_rect`、`draw_line`、`draw_ellipse`、`draw_polygon`、`draw_polyline`、`draw_path`、`draw_text`、`draw_vector_composition` 和 `change_stroke` 承载。
- 选择与变换：框选、套索、连续区域选择、移动、缩放、旋转、倾斜、裁剪、对齐、分布、重排、成组。当前已实现按 `object_id`、类型、颜色、最后对象或全部对象进行语音选择，并支持 `select_by_region` 矩形区域框选、`select_by_lasso` 多边形套索选择、`select_similar` 按填充/描边/类型/语义类型选择相似对象；同时支持移动、缩放、旋转、改色、透明度、复制、删除、清空、撤销、重做、层级前后、对齐、均匀分布、成组、取消成组和矩形裁剪。像素级连续区域魔棒、倾斜变换和布尔运算仍是计划能力。
- 图层与素材：图层顺序、遮罩、剪贴、混合模式、导入、搜索、替换。当前通过绘制顺序、对象 `object_id`、SVG 分层、素材搜索/导入/替换、`apply_clip_mask` 剪贴遮罩和 `change_blend_mode` 混合模式部分实现；精细图层面板尚未实现。
- 图像滤镜：当前支持对 Fabric 图片对象应用亮度、对比度、模糊、灰度、反色和饱和度滤镜；复杂滤镜链、局部滤镜和非图片对象滤镜仍是计划能力。

## 4. 未完成部分与原因

### 4.1 首次麦克风授权

浏览器安全策略要求用户显式授予麦克风权限，页面无法绕过。应用会自动尝试开始持续聆听，并保留一个授权失败后的按钮回退入口；授权完成后，创作流程无需鼠标或键盘。

### 4.2 真实照片与高精度文生图

当前工具链以 Fabric.js 可编辑矢量对象为核心，没有接入光栅文生图模型。复杂场景会生成可编辑矢量插画，而非写实图片。后续应新增独立图片生成 provider、资源缓存、安全校验与画布导入工具。

分层 SVG 构图能够显著提升结构、透视、明暗和细节表现，但仍受当前语言模型 SVG 构图能力影响，不能等同于专业 3D 渲染器或光栅文生图模型。

### 4.3 执行中取消

“停止当前绘图”会使用 `AbortController` 中止浏览器请求、清空等待队列，并保证被停止请求的返回结果不会应用到画布。当前 `/api/chat` 仍是同步请求，前端在任务完成前拿不到 `run_id`，因此无法释放 daemon 内正在运行的 Agent 资源。后续应改为“提交任务 -> 返回 run_id -> WebSocket 接收结果”的异步协议，并增加按 run id 取消的服务端接口。

### 4.4 低置信度澄清

不同 Chromium 版本对 SpeechRecognition `confidence` 的实现不一致，暂未使用固定阈值阻断指令。当前通过噪声过滤、危险操作确认、TTS 反馈和“重试”降低误操作风险。后续可结合多候选识别结果与 LLM 意图置信度进行澄清。

## 5. 测试与验收

- Python 单元与集成测试：`python -m pytest -q`
- 前端语音命令测试：`cd frontend-app && npm test`
- 前端构建：`cd frontend-app && npm run build`
- Python 语法检查：`python -m compileall agent server tests -q`
- 差异检查：`git diff --check`

手工验收场景：

1. 打开页面并授权麦克风，确认页面自动进入持续聆听。
2. 连续说“画一棵树”“在旁边画一座房子”，确认两条指令顺序执行。
3. 说“撤销”“重做”，确认画布立即变化，无模型等待。
4. 说“清空画布”，确认画布未立即清空；再说“确认清空”后清空。
5. 说“画一幅细节丰富的森林插画”，确认生成多个分层元素。
6. 说“画一辆有正常结构和 3D 透视的汽车”，确认生成包含 10 个以上可见矢量元素的整体构图，并能看到不同尺度车轮、车舱、玻璃、阴影和高光。
7. 模拟失败后说“重试”，确认重新提交上一条远程绘图指令。
8. 在复杂绘图执行期间说“停止当前绘图”，确认当前结果不应用且等待队列被清空。
